# weight-keeper

Маленький бот-«сторож», который ловит не похудение, а **откат после него**.

Ты шлёшь свой вес одним числом в день через [ntfy](https://ntfy.sh/) — бот
отвечает зоной: 🟢 держишься у своего лучшего · 🟡 пополз вверх · 🔴 разворот.
Никакого мессенджера, аккаунтов и облака: труба ntfy + текстовые файлы на твоей VPS.

## Зачем

Большинство трекеров веса хорошо считают **спуск** к цели, но слепы к тому, что
бывает потом: медленному отъезду обратно на месяцы и годы. Этот бот — про вторую
фазу. Его сигнал не «линия к цели», а **храповик от твоего лучшего сглаженного
веса**: пол едет вниз сам с каждым новым минимумом, а вот рост над ним меряется
как просадка. Ползучий набор +0.3 кг/мес покраснеет задолго до того, как
превратится в потерянный год.

## Особенности

- **Ноль зависимостей.** Только системный Python 3.9+. `git clone` — и работает.
- **База = бэкап.** Хранит данные в формате приложения
  [Monitor Your Weight](https://monitoryourweight.com/) байт-в-байт — файл бота
  можно импортировать обратно в приложение.
- **EMA по реальному времени** — пропуски (отпуск без весов) не врут.
- **Грейс после паузы** — первое число после перерыва не паникует на воде/соли.
- **Асимметричное трение.** Пол вниз — автоматически. Пол вверх (признать новую
  норму) — только руками на хосте, с письменной причиной. Уступка должна стоить усилия.
- **Прозрачность.** Каждый ответ показывает свою арифметику (EMA · пол · просадка
  · тренд). Всё состояние — в человекочитаемых файлах, любую зону можно
  пересчитать на бумаге.

## Быстрый старт

```bash
git clone <repo-url> weight-keeper
cd weight-keeper
cp .env.example .env && nano .env      # URL ntfy, юзер/пароль, топики
chmod 600 .env

cp weight.txt.example weight.txt       # попробовать на синтетике (или подложить свой экспорт)
python3 bot.py --cold-start            # построить состояние из истории
python3 bot.py                         # запустить демон (обычно через systemd)
```

Полная установка, systemd, ручной разбор внутренностей и troubleshooting —
в [`runbook.md`](runbook.md). Шпаргалка по командам — [`help.md`](help.md).

## Команды (шлются в топик ввода ntfy)

| Сообщение | Действие |
|-----------|----------|
| `64.7`    | записать вес за сегодня, вернуть зону |
| `status`  | текущая зона + последняя запись, без новой записи |
| `undo`    | отменить последнюю запись за сегодня |
| `help`    | прислать шпаргалку файлом |

## Файлы

| | |
|---|---|
| `bot.py` | сеть (ntfy), главный цикл, диспетчер команд |
| `keeper.py` | логика: формат, EMA, храповик, зоны (без сети) |
| `config.json` | пороги зон и периоды — правятся без знания Python |
| `runbook.md` / `help.md` | руководство и шпаргалка |
